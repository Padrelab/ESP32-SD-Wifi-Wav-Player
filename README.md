# ESP32 SD WiFi WAV Player

Минимальный проект на `ESP-IDF` для `ESP32-S3 Super Mini`, который:

- бесконечно по кругу воспроизводит `WAV` из корня SD-карты
- поднимает `Wi-Fi SoftAP`
- позволяет по веб-интерфейсу заменить `WAV`
- позволяет менять громкость воспроизведения по веб-интерфейсу
- автоматически пытается восстановить `SD` после временных сбоев, пропадания карты и её возврата
- позволяет по веб-интерфейсу загрузить новый firmware `.bin` и прошить устройство по `OTA`

Проект использует и упрощает уже готовые низкоуровневые части из
`/Users/padremac/Documents/GitHub/audio-engine-esp-idf`:

- `WAV parser`
- `PCM float32 -> int32`
- `I2S output`
- файловый stream abstraction

## Что сейчас предполагается

- целевой чип: `ESP32-S3`
- аудиовыход: внешний `I2S DAC / I2S amplifier`
- SD-карта: через `SPI`
- основной файл: `/sdcard/loop.wav`

Если у вашей платы или разводки другие GPIO, поменяйте их через `idf.py menuconfig`.

## Пины по умолчанию

### SD SPI

- `MOSI = GPIO11`
- `MISO = GPIO13`
- `SCLK = GPIO12`
- `CS   = GPIO10`

### I2S

- `DOUT = GPIO4`
- `BCLK = GPIO5`
- `WS   = GPIO6`
- `MCLK = disabled`

Эта раскладка держит все рабочие сигналы в диапазоне `GPIO1-GPIO13`.
`SD SPI` сидит на `GPIO10-GPIO13`, а `I2S` на `GPIO4-GPIO6`.
Я бы по-прежнему избегал `GPIO48` из-за встроенного RGB LED, а также не трогал `TX/RX`, `GPIO45` и `GPIO46`, если в этом нет явной необходимости.

```bash
source /Users/padremac/esp/esp-idf/export.sh
idf.py set-target esp32s3
idf.py build
```

## Первая прошивка

```bash
idf.py -p PORT flash monitor
```

После старта устройство:

- поднимает SoftAP `ESP32-WAV-Player`
- пароль по умолчанию: `wavplayer`
- веб-интерфейс доступен по адресу `http://192.168.4.1/`

## Работа с WAV

Вариант 1:

- положить `loop.wav` в корень SD-карты вручную

Вариант 2:

- подключиться к Wi‑Fi устройства
- открыть `http://192.168.4.1/`
- выбрать `.wav`
- нажать `Upload WAV`

После загрузки файл на SD будет заменён, а воспроизведение перезапустится с начала нового файла.

## Устойчивость SD-карты

Плеер старается автоматически переживать временные проблемы с `SD`:

- при transient-сбое чтения или потере доступа к файловой системе запускается `forced remount`
- повторные неудачи идут с `exponential backoff`
- при повторных recovery-сбоях частота `SD SPI` автоматически снижается вплоть до `4 MHz`
- если recovery удался на пониженной частоте, эта более безопасная частота защёлкивается до перезагрузки
- после нескольких активных попыток плеер переходит в `waiting_for_media` и периодически опрашивает карту
- после возврата карты и успешного remount воспроизведение автоматически запускается снова

В `/api/status` дополнительно видны:

- `state: recovering` или `state: waiting_for_media`
- `sd_remount_count`
- `sd_remount_failure_count`
- `sd_recovery_backoff_ms`

## Громкость

В веб-интерфейсе есть ползунок `Volume`. Он меняет громкость сразу, без перезапуска текущего трека.
Значение хранится в `NVS`, поэтому переживает перезагрузку платы.

Также доступны простые HTTP endpoint:

- `GET /api/volume` возвращает текущий процент громкости
- `POST /api/volume` принимает число `0..100` в теле запроса

## OTA обновление

1. Соберите новый firmware:

```bash
idf.py build
```

2. В веб-интерфейсе:

- выберите `build/esp32_sd_wifi_wav_player.bin`
- нажмите `Upload Firmware`

После успешной загрузки прошивка будет записана в OTA-раздел, выбран новый boot partition и плата автоматически перезагрузится.

## Ограничения текущей версии

- поддерживаются `WAV PCM 16/24/32-bit` и `float32`
- поддерживаются только `mono` и `stereo`
- логика рассчитана на один фиксированный файл `loop.wav`
- режим Wi‑Fi сейчас только `SoftAP`, без подключения к внешней сети
